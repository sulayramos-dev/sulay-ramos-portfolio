# Case Input Structure

Gebruik deze map om per portfolio-case extra context, screenshots en publicatie-notities te verzamelen.

## Cases

1. `01-secure-otap-home-infrastructure`
2. `02-ai-driven-operations-readiness`
3. `03-oase-innovatieplatform`
4. `04-crm-systeem`

## Screenshot regels

- Plaats screenshots per case in de map `screenshots/`.
- Gebruik duidelijke bestandsnamen, bijvoorbeeld `01-dashboard-redacted.png`.
- Blur of verwijder klantnamen, e-mails, telefoonnummers, IP-adressen, interne hostnames, interne tickets, notities en IDs.
- Voeg in de case README toe wat elk screenshot laat zien.

## Aanbevolen context per case

- Probleem of aanleiding
- Doelgroep/gebruiker
- Jouw bijdrage
- Technische keuzes
- Proces- of architectuurkeuzes
- Resultaat of impact
- Wat niet publiek gedeeld mag worden
